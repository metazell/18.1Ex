- What is HTTP?
  • HTTP (HyperText Transfer Protocol) is the protocol used for transmitting hypertext requests and information on the internet. It is the foundation of any data exchange on the Web and it is a protocol used for fetching resources such as HTML documents. HTTP follows a client-server model where the client (such as a web browser) makes a request to the server, which then responds.

      • - What is DNS?
  • DNS (Domain Name System) is the system used to translate human-friendly domain names (like www.example.com) into IP addresses that computers use to identify each other on the network. DNS is like the phonebook of the internet, converting easy-to-remember names into the numerical IP addresses needed for locating and identifying computer services and devices.
      • - What is a query string?
  • A query string is a part of a URL that assigns values to specified parameters. It is typically used to pass data to web applications. A query string begins with a question mark (?) followed by key-value pairs separated by '&'.
      • - What are two HTTP verbs and how are they different?
  • GET: Used to request data from a specified resource. GET requests are typically used to retrieve data and have no effect on the data. POST: Used to send data to a server to create/update a resource. POST requests can change the state of the server and typically involve sending data in the body of the request.
      • - What is an HTTP request?
  • HTTP request is a message sent by the client to the server to request data or perform an action.
      • - What is an HTTP response?
  • HTTP response is a message sent by the server to the client in reply to an HTTP request.
      • - What is an HTTP header? Give a couple examples of request and response headers you have seen.
  • HTTP headers are key-value pairs sent in HTTP requests and responses to provide information about the request or response or about the object sent in the message body.
      • - What are the processes that happen when you type “http://somesite.com/some/page.html” into a browser?
  • Resolution: The browser contacts a DNS server to resolve the domain name (somesite.com) into an IP address.
  • TCP Connection: The browser establishes a TCP connection with the server at the resolved IP address.
  • HTTP Request: The browser sends an HTTP request to the server, requesting the specific resource (/some/page.html).
  • Server Processing: The server processes the request, finds the requested resource, and prepares an HTTP response.
  • HTTP Response: The server sends the HTTP response back to the browser, including the requested resource (e.g., an HTML page).
  • Rendering: The browser renders the HTML content, potentially making additional requests for resources like images, CSS, and JavaScript files as specified in the HTML.
